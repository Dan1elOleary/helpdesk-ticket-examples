# Slow Computer Investigation Example

## Ticket Title

User reports slow workstation performance

---

## Ticket Information

| Field | Details |
|---|---|
| User | Example User |
| Device Name | DESKTOP-001 |
| Operating System | Windows 11 |
| Issue Type | Performance |
| Priority | Medium |
| Status | Resolved |

---

## Issue Summary

The user reported that their workstation was running slowly. Applications were taking longer than usual to open, and general system performance was degraded.

---

## User Impact

The issue affected the user’s productivity by slowing down access to applications, files, and daily work tasks.

---

## Environment

- Windows 11 workstation
- Standard user profile
- Company-managed endpoint
- Microsoft 365 applications
- Endpoint security software installed

---

## Troubleshooting Steps

1. Confirmed the performance issue with the user.
2. Asked when the issue started and whether it occurred after any recent changes.
3. Checked system uptime.
4. Reviewed CPU, memory, and disk usage in Task Manager.
5. Checked available disk space.
6. Reviewed startup applications.
7. Checked for pending Windows updates.
8. Restarted the workstation.
9. Verified that endpoint protection was running.
10. Checked for obvious unwanted or unnecessary applications.
11. Cleared temporary files where appropriate.
12. Confirmed improved performance with the user.

---

## Findings

The workstation had high uptime and multiple background applications running. Available disk space was sufficient, and no immediate signs of malware or critical system errors were observed.

After restarting the workstation and reviewing startup applications, performance improved.

---

## Root Cause

The issue was likely caused by extended system uptime and multiple background processes consuming system resources.

---

## Resolution

Restarted the workstation, reviewed resource usage, reduced unnecessary startup impact where appropriate, and confirmed improved performance with the user.

---

## User Communication

The user was informed that the workstation had been running for an extended period and that a restart helped clear background resource usage.

The user was advised to restart the workstation regularly to help maintain performance.

---

## Follow-Up Notes

If the issue returns, additional troubleshooting may include:

- Reviewing Event Viewer logs
- Running hardware diagnostics
- Checking disk health
- Reviewing endpoint security alerts
- Checking user profile health
- Evaluating whether hardware upgrades are needed

---

## Closure Summary

Investigated slow workstation performance, reviewed system resource usage, restarted the device, and confirmed improved performance with the user. Ticket resolved.
